# moneyman (invoices-bot)

Scrapes Israeli bank/credit card accounts via `israeli-bank-scrapers` and exports transactions to multiple storage backends. Runs as a scheduled GitHub Actions job or locally via Docker/Node.

## Commands

```bash
npm run build          # tsc → dst/
npm run start          # node dst/index.js
npm test               # jest (ts-jest, tests in src/)
npm run lint           # prettier --check
npm run lint:fix       # prettier --write
npm run test:config    # build + verify config schema
npm run start:container  # docker compose up
```

Debug logging:
```bash
DEBUG=moneyman:* npm run start
```

## Architecture

```
src/
  index.ts              # entry point: orchestrates scrape → store
  config.ts             # parses MONEYMAN_CONFIG / MONEYMAN_CONFIG_PATH (JSONC, Zod validation)
  config.schema.ts      # Zod schema for full config
  types.ts              # shared TypeScript types
  runnerMetadata.ts     # external IP logging, run metadata
  scraper/              # wraps israeli-bank-scrapers, handles OTP/browser
  bot/
    index.ts            # runWithStorage: wires scraper results to all storage backends
    notifier.ts         # Telegram error/status notifications
    storage/            # one file per backend: sheets, sql, ynab, buxfer, actual,
                        #   azure-data-explorer, telegram, json, web-post, monday
  security/             # domain firewall (per-scraper allow/block rules)
  utils/                # logger, secure-log, currency, promises, timer, jsonc
  scripts/              # verify-config.ts (standalone validation)
dst/                    # compiled JS output (gitignored, regenerated by tsc)
```

## Configuration

Config is provided via environment variable or file — never hardcoded:

| Variable | Description |
|---|---|
| `MONEYMAN_CONFIG` | Inline JSON/JSONC string |
| `MONEYMAN_CONFIG_PATH` | Path to a `.json` / `.jsonc` file |
| `MONEYMAN_UNSAFE_STDOUT` | `"true"` to print logs to stdout (default: `"false"`, logs go to `/tmp/moneyman.log`) |
| `MONEYMAN_LOG_FILE_PATH` | Override log file path (default: `/tmp/moneyman.log`) |
| `TZ` | Timezone (default: `Asia/Jerusalem`) |
| `DEBUG` | Set to `moneyman:*` for verbose debug output |

Local config files (`moneyman.conf.jsonc`, `moneyman.invoices.conf.jsonc`) are gitignored — point `MONEYMAN_CONFIG_PATH` at them.

See `config.example.jsonc` for a full annotated example.

## Gotchas

- **Node >= 24 required** — enforced in `package.json` engines
- **ESM project** (`"type": "module"`) — use `.js` extensions in all imports, even for `.ts` source files
- **Compiled output in `dst/`** — always `npm run build` before `npm run start` or `test:config`
- **Logs are redirected by default** — when `MONEYMAN_UNSAFE_STDOUT=false` (default), stdout is piped to the log file; `logToPublicLog()` bypasses this and always appears in Docker logs
- **Pre-commit hook** runs `pretty-quick --staged` via Husky — commits will fail if formatting is off; run `lint:fix` first
- **Jest module mapper** strips `.js` extensions for test resolution — tests import without extensions

## Testing

```bash
npm test                          # all unit tests
npm run test:config               # validates your local config file
npm run test:scraper-access       # live scraper connectivity check (requires config)
```

Tests live alongside source in `src/` as `*.test.ts`. Use `pg-mem` for SQL storage tests (no real DB needed).

## Storage Backends

Each backend is opt-in via the `storage` config key: `googleSheets`, `sql`, `ynab`, `buxfer`, `actual`, `azure`, `telegram`, `localJson`, `webPost`, `monday`.

Adding a new backend: create `src/bot/storage/<name>.ts`, export a class/function matching the storage interface, register it in `src/bot/storage/index.ts`.
